# Requirement Analysis Prompt

Read:
- `.ai-workflow/00-user-request.md`
- `AGENTS.md`

If the user provided a new product request in chat, save it to `.ai-workflow/00-user-request.md`.

Do not write code.

Create:
- `.ai-workflow/01-requirement-analysis.md`

The document must include product goal, target users, pain points, scenarios, user stories, must-have features, nice-to-have features, out of scope, business rules, edge cases, risks, unknowns, review questions, and approval checklist.

Stop after writing the document.
Ask the user to approve or request changes.
