## 目标

构建一个可在 bbBot 中使用的星露谷攻略检索工具，支持命令和 AI tool 自然语言调用。最终目标不是只覆盖固定命令，而是尽量覆盖玩家游玩过程中临时想到的攻略问题。

## 当前实现

### 入口

- 聊天命令：`/星露谷 ...` / `星露谷 ...` / `/stardew ...`
- AI tool：`stardew_guide`

### 检索链路

`/星露谷` 命令和 `stardew_guide` AI tool 统一走同一条链路：

1. `StardewQueryPlannerService` 调用 `AiChatService` LIGHT 档，把用户问题规划为 1-4 个 intent。
2. 每个 intent 包含类型、类型内检索关键词，以及季节、地点、天气、时间、居民等约束。
3. `StardewGuideRetriever` 按类型追加检索 hint，调用 typed evidence API 获取证据。
4. `StardewGuideAssistantService` 用 `AiChatService` CHAT 档整合成自然回复。
5. 本地结构化知识库未命中时，走官方中文 Wiki API 搜索和摘要兜底。

### 重要设计结论

- `@AiTool.description` 只负责让星露谷攻略问题进入工具，不承载内部分类清单。
- 分类和关键词抽取放在 `StardewQueryPlannerService`，证据检索放在 `StardewGuideRetriever`，自然回复放在 `StardewGuideAssistantService`。
- `StardewGuideService.answerEvidence(intent, query)` 是 typed evidence API；旧 `answer(String)` 只保留为兼容入口。
- typed plan 无证据时不再无条件回退自由文本路由，降低“用户问 A、旧规则答 B”的风险。
- handler 和 AI tool 回复不展示数据版本、校验日期、来源链接、Wiki 或本地库等内部信息。
