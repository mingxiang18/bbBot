# 星露谷攻略助手实现结论

> 阶段：Implementation progress
> 日期：2026-06-26
> 分支：`master`

## 当前结论

星露谷攻略助手已经形成一版可发布基础能力：`/星露谷` 命令和 `stardew_guide` AI tool 走同一条链路，由 AI 做问题分类和关键词抽取，再按类型检索结构化证据，最后整合成自然语言回复。回复层不展示数据版本、校验日期、Wiki、本地库等内部实现信息。

这不是完整百科，但已经覆盖高频游玩问题，并通过 Wiki 兜底减少“用户临时想到的问题完全答不上”的风险。后续重点不是继续堆固定命令，而是补强高价值结构化数据、typed 检索边界和回归测试。

## 已实现链路

1. `StardewQueryPlannerService` 调用 `AiChatService` LIGHT 档，将用户问题规划为 1-4 个 typed intent。
2. 每个 intent 包含类型、类型内关键词，以及季节、地点、天气、时间、居民等约束。
3. `StardewGuideRetriever` 按类型追加检索 hint，并调用 typed evidence API。
4. `StardewGuideAssistantService` 调用 `AiChatService` CHAT 档，把证据整合为自然回复。
5. 本地结构化知识库未命中时，使用官方中文 Wiki API 搜索和摘要兜底。

关键设计结论：

- `@AiTool.description` 只负责让星露谷攻略问题进入工具，不承载内部分类清单。
- 分类和关键词抽取放在 planner，证据检索放在 retriever，自然语言回复放在 assistant service。
- `StardewGuideService.answerEvidence(intent, query)` 是 typed evidence API；旧 `answer(String)` 只保留兼容。
- typed plan 无证据时不再无条件回退自由文本路由，避免“问 A 答 B”的测试命中地狱。
