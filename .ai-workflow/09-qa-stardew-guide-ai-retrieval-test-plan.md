# 星露谷攻略 AI 检索链路测试方案

> 阶段：QA test plan
> 日期：2026-06-26
> 分支：`master`
> 范围：`/星露谷` 命令入口、`stardew_guide` AI tool、AI 分类计划、typed evidence 检索、证据整合、自然语言回复
> 明确排除：不做存档解析，不读取或推断玩家本地存档

## 1. 测试目标

本轮改造的核心不是继续堆确定性命令分支，而是让命令入口和 AI tool 统一成可扩展的攻略问答路径：

1. 用户自然语言输入问题。
2. AI 将问题拆成 1-4 个 typed intent，并抽取适合检索的中文关键词、季节、地点、天气、时间、居民等条件。
3. `StardewGuideRetriever` 只按 typed plan 调用 `StardewGuideService.answerEvidence(type, query)` 获取结构化证据。
4. `StardewGuideAssistantService` 根据资料整合成自然、简洁、可执行的中文回复。
5. 回复不展示内部字段，例如数据版本、校验日期、来源链接、Wiki、本地库、证据编号或关键词。

测试需要证明两个方向：

- 功能方向：常见玩家问题能被检索、整合、自然回答。
- 稳定方向：AI 输出格式松散、AI 为空、AI 抛异常、问题含歧义时不会直接失败或胡说；typed miss 不会偷偷回落到旧自由文本路由导致问 A 答 B。

## 2. 自动化测试范围

### 2.1 `StardewGuideAssistantServiceTest`

覆盖 AI 编排层，即 `/星露谷` 命令和 `stardew_guide` tool 共用的主路径。

必测用例：

- JSON typed plan：AI 返回 JSON 时，服务应解析 `intents[].type`、`keywords`、`constraints`，再交给 retriever 检索。
- Planner 不可用：LIGHT 模型返回空、异常或不可解析 JSON 时，应使用本地高置信 fallback plan，而不是调用旧自由文本 `StardewGuideService.answer`。
- 分类职责边界：AI 返回具体非 `GUIDE` 类型时以 AI 分类为主；本地规则只纠偏高风险串路，例如通用 `GUIDE` 被拉回明确的 `CRAFTING`、`FESTIVAL`、`FARM_MAP` 或 `ANIMAL_CARE`，以及农场地图与农场建筑、农场动物与农场建筑互相误判。
- 证据整合 prompt：最终整合 prompt 必须包含用户问题和 typed evidence，且不携带 `sourceUrls`、`gameVersion`、`lastCheckedAt` 这类内部字段。
- 自然回复返回：当 CHAT 模型返回非空答案时，命令层直接使用该自然语言答案。
- CHAT 空返回：应退回第一条 typed evidence 的原文答案；如果没有可靠证据，则给出补充条件提示。
- 发布路径边界：Assistant 不应调用旧 `StardewGuideService.answer(String)`；旧入口只保留兼容和历史回归。
- 关键词契约：获取、制作、升级、购买、位置、收集包类问题的 `keywords` 应保留动作，例如“怎么获得”“怎么做”“升级材料”“在哪里”，避免只用裸物品名降低检索质量。

代表问题：

- `星露谷 恐龙蛋黄酱怎么弄，电影院要用`
- `星露谷 矮人卷轴在哪刷`
- `星露谷 电池组怎么获得`
- `星露谷 斧头升级需要什么`
- `星露谷 海滩农场洒水器能用吗`

### 2.2 `BbStardewHandlerTest`

覆盖命令入口。

必测用例：

- `/星露谷` 或 `星露谷` 命令应调用 `StardewGuideAssistantService.answer(message.getMessage())`。
- 回复内容应为 AI 整合后的自然答案。
- 回复不得包含 `来源：`、`数据版本`、`校验日期`、`stardewvalleywiki.com` 等内部展示。
- handler 异常分支保留兜底错误提示，便于线上排查。

### 2.3 `StardewGuideToolTest`

覆盖 AI tool 输出契约。

必测用例：

- `stardew_guide` tool 返回 `query`、`intent`、`evidence`、`replyInstruction`。
- tool 不返回 `sourceUrls`、`gameVersion`、`lastCheckedAt`、`note` 等容易被模型直接复述给用户的内部字段。
- `replyInstruction` 明确要求 AI 使用资料自然回答，不要展示来源、版本、日期、Wiki、本地库等字样。
- tool 在典型问题下能给出足够资料供上层模型整合。

代表问题：

- `夏天雨天海边能钓什么鱼`
- `失踪的收集包需要什么`
- `海莉今天下午两点在哪里`
- `恐龙蛋怎么获得`
- `斧头升级要多少钱和材料`

### 2.4 `StardewGuideServiceTest`

覆盖本地 typed evidence 和历史兼容回归。新发布路径应优先新增 `answerEvidence(type, query)`、`StardewGuideRetrieverTest` 和 `StardewGuideAssistantServiceTest`，旧 `answer(String)` 只用于保护存量行为。

必测领域：

- 鱼类：季节、天气、地点、时间、蟹笼、夜市、矿井、姜岛、传说鱼。
- 收集包：具体小包、房间概览、金库金额、失踪的收集包。
- 居民：34 位居民均有本地日程；有时间能回答位置；缺少游戏内时间时要求补条件。
- 礼物：生日、最爱礼物、喜欢礼物、通用送礼规则。
- 资源：硬木、电池组、恐龙蛋、恐龙蛋黄酱、矮人卷轴、兔子的脚、红叶卷心菜、鱼籽酱、龙牙等。
- 工具：斧头、镐子、喷壶、锄头、垃圾桶、鱼竿的升级或购买条件。
- 建筑：鸡舍、畜棚、筒仓、鱼塘、马厩、房屋升级等。
- 制作配方：完整制作页 150 条，覆盖栅栏、种子、地板/路径、照明、家具、标牌、精炼设备、工匠设备、肥料、图腾、戒指、钓具和杂项。
- 机器兼容：小桶、罐头瓶、蛋黄酱机、鱼熏机、脱水机、诱饵制造机等旧 `MACHINE` 路径仍能回答。
- 商店：皮埃尔、铁匠、罗宾、玛妮、威利、科罗布斯、旅行货车、沙漠商人、书商。
- 节日/活动：全年 12 个节日/活动，覆盖日期、入场时间、地点、玩法、奖励、商店/兑换重点、是否时间流逝、店铺是否关闭、动物是否要喂。
- 农场地图：8 张开局农场地图，覆盖特点、适合人群、可耕地/可建造空间、特殊资源、限制、钓鱼水域和选择建议。
- 农场动物：11 个鸡舍/畜棚动物，覆盖购买/获得方式、成熟时间、产物频率、采集工具、加工产物、好感/心情/运气/天气机制和赚钱建议。
- 料理和 buff：幸运、速度、钓鱼、采矿、战斗、骷髅洞穴推荐。
- 技能与进度：五大技能升级、职业选择、沙漠、温室、骷髅洞穴、博物馆、精通系统。

重点回归边界：

- `金钱` 不能误判为 `金斧` 或金级工具。
- `矮人卷轴在哪刷` 不能被居民“矮人”抢走。
- `恐龙蛋怎么获得` 不能被大鸡舍建筑抢走。
- `橡树树脂怎么获得，小桶缺这个` 应优先回答资源获取。
- `三种果冻怎么钓` 和 `海果冻怎么获取` 应继续走钓鱼收藏物。
- `冬季矿井能钓什么鱼` 中的“井”不能误判为建筑“水井”。
- `海滩农场洒水器能用吗` 应走 `FARM_MAP`，不能误判为建筑或普通攻略。
- `农场建筑有哪些`、`鸡舍升级材料` 应走 `BUILDING`，不能误判为 `FARM_MAP`。
- `兔子的脚怎么出`、`猪松露怎么赚钱` 应走 `ANIMAL_CARE` 的动物详情，不能退成动物列表或资源泛答。
- `后期养什么动物赚钱` 应走 `ANIMAL_CARE` 的动物对照列表，并突出猪、松露和松露油。
- `鸡舍升级材料`、`畜棚多少钱` 应走 `BUILDING`，不能被农场动物抢走。

## 3. 场景矩阵

### 3.1 钓鱼

- `夏天能钓什么鱼`
- `夏季雨天海边能钓什么`
- `春天晚上河里有什么鱼`
- `冬季矿井能钓什么鱼`
- `夜市潜艇能钓什么`
- `姜岛能钓什么`
- `五条传说鱼在哪里钓`
- `蟹笼能抓什么`
- `三种果冻怎么钓`

验收点：

- 季节、天气、地点、时间要在答案里体现。
- 条件不足时说明缺少条件。
- 不把建筑或资源路由误命中为鱼类问题。

### 3.2 收集包

- `春季作物收集包需要什么`
- `海鱼收集包交什么`
- `布告栏染料收集包需要什么`
- `金库一共要多少钱`
- `失踪的收集包需要什么`
- `电影院收集包缺恐龙蛋黄酱怎么办`

验收点：

- 列出关键物品或金额。
- 房间概览问题能回答该房间包含的小包。
- 失踪的收集包应能关联电影院。

### 3.3 居民

- `海莉今天下午两点在哪里`
- `亚历克斯夏天周三 10 点在哪`
- `罗宾下雨天上午在哪里`
- `科罗布斯晚上在哪`
- `桑迪生日喜欢什么`
- `莉亚最爱礼物是什么`
- `海莉在哪里`

验收点：

- 有游戏内时间的问题尽量给位置和找人建议。
- 没有游戏内时间的问题应提示补充时间，不要装作能知道“当前时间”。
- 礼物问题不应被位置路由抢走。
- 不声称读取存档、实时游戏状态或玩家当前日期。

### 3.4 资源与稀有物品

- `硬木怎么刷`
- `电池组怎么获得`
- `恐龙蛋怎么获得`
- `恐龙蛋黄酱怎么做`
- `矮人卷轴在哪刷`
- `兔子的脚怎么获得`
- `红叶卷心菜怎么获得`
- `龙牙在哪里刷`
- `鱼籽酱怎么做`

验收点：

- 给出主要来源、可执行路线和阶段建议。
- 稀有物品要给保守建议，避免承诺必出。
- 和居民、建筑、机器同名或近名时优先回答用户实际问法。

### 3.5 工具、建筑、机器、商店

- `斧头升级需要什么条件和钱`
- `钢斧要多少钱`
- `铱镐材料是什么`
- `喷壶先升到几级合适`
- `鸡舍升级材料`
- `筒仓要不要先建`
- `鱼熏机怎么做`
- `小桶怎么解锁`
- `木栅栏怎么做`
- `茶苗材料是什么`
- `树液采集器配方`
- `迷你锻造台怎么做`
- `栅栏有哪些`
- `照明有哪些`
- `铱制洒水器在哪里买`
- `书商什么时候来`

验收点：

- 金额、材料、地点、耗时、顺序限制清楚。
- `金钱` 不误判为金级工具。
- 制作菜单里的配方问题归 `CRAFTING`；旧 `MACHINE` 只作为兼容入口。
- 机器加工产物和资源交叉问题应按动词判断，例如“树脂怎么获得”优先资源，“小桶怎么做”优先制作配方，“恐龙蛋黄酱怎么做”优先资源获取/加工方式。

### 3.6 节日/活动

- `春季节日有哪些`
- `沙漠节怎么玩，三花蛋换什么`
- `花舞节几点开始`
- `夏威夷宴会放什么进汤`
- `鳟鱼大赛金色标签怎么用`
- `星露谷展览会怎么拿星之果实`
- `万灵节迷宫奖励是什么`
- `冰雪节钓鱼比赛怎么赢`
- `鱿鱼节奖励怎么算`
- `夜市潜艇几点关`
- `冬星盛宴送什么`

验收点：

- 节日本身问题归 `FESTIVAL`，回答日期、入场时间、地点、玩法、奖励和注意事项。
- `沙漠节`、`鳟鱼大赛`、`鱿鱼节`、`夜市` 要说明时间继续流逝、普通店铺开放、动物仍要喂。
- 明确具体商品购买问题仍可归 `SHOP`，例如“草莓种子在哪里买”“万灵节稀有稻草人多少钱”。
- 不把“花舞节几点开始”误路由成居民日程。

### 3.7 农场地图/布局选择

- `新手选什么农场`
- `海滩农场洒水器能用吗`
- `草原农场开局有什么`
- `森林农场硬木怎么样`
- `河流农场适合钓鱼吗`
- `四角农场适合多人吗`
- `哪种农场不能大面积用洒水器`
- `农场建筑有哪些`
- `鸡舍升级材料`

验收点：

- 农场选择、地图特点、适合人群、特殊资源和限制归 `FARM_MAP`。
- 农场建筑、建造价格、升级材料和罗宾建造归 `BUILDING`。
- 推荐型问题要能按字段命中，例如新手、动物、钓鱼、采矿、硬木、多人、战斗、洒水器，而不是只靠固定命令短语。
- 不展示 Wiki、数据版本、校验日期等内部信息。

### 3.8 农场动物

- `动物有哪些`
- `前期养什么动物好`
- `后期养什么动物赚钱`
- `兔子的脚怎么出`
- `鸭毛概率高吗`
- `奶牛为什么不出大壶牛奶`
- `猪松露怎么赚钱`
- `鸵鸟蛋拿到后怎么办`
- `金鸡怎么获得`
- `鸡舍升级材料`

验收点：

- 单个动物或动物产物问题归 `ANIMAL_CARE` 动物详情，回答建筑、获得方式、成熟时间、产物、加工和关键机制。
- 类目推荐问题归动物对照列表，能展示“产物->加工物”，例如松露到松露油、鸵鸟蛋到蛋黄酱。
- 通用养殖问题仍可回到 `guide`，例如“动物怎么养、怎么提高心情”。
- 鸡舍/畜棚升级材料和价格归 `BUILDING`，不能被动物数据抢走。

### 3.9 料理、技能、进度

- `骷髅洞穴吃什么料理`
- `幸运午餐怎么做`
- `香辣鳗鱼材料和效果`
- `战斗技能怎么快速升级`
- `耕种 5 级选什么职业`
- `温室怎么修`
- `沙漠怎么解锁`
- `博物馆捐赠有什么用`

验收点：

- 料理要包含材料、来源、buff 和使用场景。
- 技能要包含经验来源、升级路线、职业建议。
- 进度类答案要可执行，避免只给泛泛解释。

### 3.10 模糊与组合问题

- `电影院还差几个东西，恐龙蛋黄酱怎么弄`
- `小桶缺橡树树脂，树脂哪里来`
- `我夏天要赚钱种什么`
- `雨天要干嘛`
- `刚开局先做什么`
- `矮人相关的东西怎么解锁`

验收点：

- AI 关键词应拆成多个检索词。
- 最终回复要整合多个证据，而不是机械拼接。
- 条件不足时给可执行默认建议，并说明需要补充哪些游戏内条件。

## 4. 人工验收方案

人工验收用于验证真实 bot、真实 AI 输出和用户体验。

前置条件：

- Docker 已启动。
- `misu-mysql-local` 容器运行中，端口 `3316->3306` 可用。
- mock OpenAI 或真实 AI 配置可用。
- 当前目标版本已完成单元测试和编译。

启动命令：

```bash
./scripts/dev/bb-dev.sh up --build
```

基础验收：

1. 在聊天入口发送 `星露谷 斧头升级需要什么条件和钱`。
2. 期望回复自然说明铜/钢/金/铱升级材料、金币、铁匠铺和顺序限制。
3. 回复不出现“来源”“数据版本”“校验日期”“Wiki”“本地库”“证据”“关键词”。
4. 发送 `星露谷 恐龙蛋黄酱怎么弄，电影院要用`。
5. 期望回复说明恐龙蛋来源、孵化或蛋黄酱机路线、电影院用途。
6. 发送 `星露谷 海滩农场洒水器能用吗`。
7. 期望回复说明海滩农场沙地限制、可用洒水器区域和适合人群。
8. 发送 `星露谷 海莉在哪里`。
9. 期望回复要求补充游戏内时间，而不是臆造当前位置。

AI tool 验收：

1. 通过普通 AI 问答触发 `stardew_guide`。
2. 观察模型最终答复是否自然整合 tool evidence。
